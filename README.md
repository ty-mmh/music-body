# music-body

これは、音楽を置くためのリポジトリです。  
ただし、ここでいう音楽は、完成した音源だけを指しません。

ここでは音楽を、  
モチーフ、記譜、構造、時間、反復、差異、緊張、解放、  
そしてそれが耳で聴かれるための仕組みまで含んだものとして扱います。

音になる前の音楽。  
記譜された音楽。  
構造としての音楽。  
Web 上で再生される音楽。  
そのすべてを、ここではひとつの連続体として置いていきます。

このリポジトリは、私と ChatGPT の共作によって育てられます。  
人間の感覚、思考、美学、モチーフの発見。  
AI による言語化、構造化、記述、実装。  
その往復のなかで、小さな音楽と、その音楽を聴くための身体としての Web アプリケーションを積んでいきます。

ここにあるのは、単なる曲集ではありません。  
単なるコード集でもありません。  
これは、音楽そのもの、音楽を鳴らす身体、その背後にある思考を、  
GitHub の上に少しずつ積層していくための場です。

## What is here

このリポジトリでは、次のようなものを扱います。

- まだ作品になっていない最小のモチーフ
- 小さな作品
- 作品を鳴らすための JavaScript / Web Audio のモジュール
- 再生基盤となる engine / app
- 制作中の思考や断片的なメモ
- 未整理の試作や実験

小さいものも大きいものも、ここでは同じ地面に置かれます。  
二つの音だけでできた断片も、短い作品も、試作中の再生機構も、  
仮置きではなく、ひとつの観測点として扱われます。

## Co-creation

このリポジトリは、人間と AI の共作を明示的に含んでいます。

ただしそれは、AI に制作を丸ごと委ねるという意味ではありません。  
ここで行われるのは、感覚を言葉へ、言葉を構造へ、構造を可聴化へと変換していく往復です。

- 人間が、音楽の核を見つける
- AI が、その核を記述し、構造化し、実装可能な形へ開く
- 人間が、それを読み、聴き、ずらし、選び直す
- AI が、再びそれを受けて整える

この反復そのものが、このリポジトリの制作工程です。

## Directory structure

```text
.
├─ README.md
├─ pieces/
│  ├─ <piece-name>/
│  │  ├─ README.md
│  │  ├─ motifs/
│  │  ├─ modules/
│  │  └─ assets/
│  └─ ...
├─ engine/
├─ app/
├─ notes/
├─ motifs/
└─ experiments/
```

### `pieces/`
小作品を置く場所です。  
各作品は独自の README を持ち、作品ごとの核、モチーフ、モジュールを抱えます。

### `engine/`
再利用可能な再生基盤を置く場所です。  
時間制御、音の生成、スケジューリングなど、複数作品を支える仕組みを育てます。

### `app/`
作品を実際に触れ、鳴らし、聴くための Web アプリケーションを置く場所です。  
ここでアプリは、単なる UI ではなく、**音楽を聴くための身体**です。

### `notes/`
制作と思考の断片を残す場所です。  
作品以前の考え、制作中のメモ、構造に関する言語化などを置きます。

### `motifs/`
まだ特定の作品に所属していない最小モチーフの置き場です。  
ここにあるものは、作品以前の種子です。

### `experiments/`
試作や未整理の実験を置く場所です。  
ここでの試みは、後に作品や基盤へ接続されるかもしれません。

## How to read this repository

このリポジトリは、完成品だけを上から順に眺めるためのものではありません。  
断片、作品、基盤、メモ、実験が相互に行き来しながら育っていく場です。

最小の断片を見るなら `motifs/`。  
個別の作品を見るなら `pieces/`。  
再生の仕組みを見るなら `engine/` と `app/`。  
制作の背後にある思考を見るなら `notes/`。  
途中の試みを見るなら `experiments/`。

このリポジトリは、音楽の完成を記録する場というより、  
音楽がどのように生まれ、どのような身体を持ちうるかを辿る場です。

## Orientation

ここで目指しているのは、固定された完成品だけではありません。

読める音楽。  
見える音楽。  
鳴らせる音楽。  
改変できる音楽。  
そして、音になる前からすでに音楽であるような断片たち。

それらを GitHub の上に置き、積み、接続し、育てていくこと。  
このリポジトリは、そのためにあります。
